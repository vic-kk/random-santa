# 🎁 Random Santa

Проект для автоматической жеребьёвки «Тайного Санты» на основе ответов из Google Forms.

## 🔄 Как это работает: два этапа жизни проекта

Проект живёт в двух разных состояниях, которые настраиваются отдельно. Это позволяет сначала собрать данные участников, а потом провести розыгрыш.

> [!TIP]
> Проект поддерживает npm и yarn. Во всех примерах используется npm, для yarn просто заменяйте `npm run` -> `yarn`.

### 1️⃣ Этап: Сбор данных

**🎯 Цель:** Сбор данных участников с помощью Google Form.

#### 🚀 Быстрый старт для этапа сбора данных

1. **Установите зависимости**

    ```bash
    npm i
    ```

2. **Настройте ссылку на форму**
   - Создайте Google Form для сбора данных
   - Вы можете использовать предзаполнение UID (пример в проекте его использует)
   - Укажите ссылку на неё в файле [`src/data/externalLinks.ts`](src/data/externalLinks.ts)

3. **Убедитесь, что включён режим сбора**
    
    ```bash
    npm run stage -- 1
    ```

4. **Запустите локально для проверки**

    ```bash
    npm run dev
    ```

5. **Соберите и опубликуйте проект**

    ```bash
    npm run build
    ```

   Готовая сборка из папки `docs/` публикуется на GitHub Pages.

### 2️⃣ Этап: Жеребьёвка и результаты

**🎯 Цель:** Случайным образом сопоставить участников и показать каждому его получателя.

#### 🚀 Быстрый старт для этапа розыгрыша

1. **Подготовьте данные**
   - Экспортируйте ответы из Google Forms
   - Переименуйте файл в `SANTA.csv`
   - Поместите его в папку `_local/` в корне проекта

   ```shell
   Структура после подготовки:
   ├─📂 _local/
   │  ├─ SANTA.csv # ответы из Google Forms
   │  └─ tip.txt
   └─ ...
   ```

2. **Запустите автоматическую обработку**

   ```bash
   npm run santa_auto
   ```

   Эта команда:
   - Сделает резервную копию старой жеребьёвки, если она была
   - Проведёт новую жеребьёвку на основе `SANTA.csv`
   - Создаст файл [`src/data/addresses.ts`](src/data/addresses.ts) с результатами
   - Переключит проект в режим "Результатов" (stage -- 3)
   - Соберёт проект для публикации

3. **Опубликуйте результаты**
   - Готовая сборка из папки `docs/` публикуется на GitHub Pages.

### 🔧 Ручное управление этапами

Если нужно разделить этапы (например, проверить жеребьёвку перед сборкой), используйте команды по отдельности:

```bash
# 1. Только жеребьёвка на основе файла SANTA.csv
# Существующая версия жеребьевки будет перенесена в _local/backups
npm run santa

# 2. Локальный запуск для предварительного просмотра
npm run dev

# 3. Сборка проекта для деплоя (GitHub Pages)
npm run build
```

## 🧪 Для разработки и тестирования

### Генерация тестовых данных

> [!WARNING]
> При генерации тестовых данных файл `_local/SANTA.csv` будет полностью перезаписан. Убедитесь, что важные данные сохранены.

```bash
# Полный цикл: генерация 15 записей + жеребьёвка
npm run mock_auto

# Только генерация CSV
npm run mock_csv        # по умолчанию, 15 записей
npm run mock_csv -- N   # N записей

```

### Смена этапов

```bash
# Выбор этапа через диалог и сборка для публикации
npm run stage_auto

# Только смена этапа для локального теста
npm run stage        # Выбор этапа через диалог
npm run stage -- N   # Явное указание этапа (1 = сбор, 2 = сервис, 3 = результаты)

```

## 📋 Справочник по командам (NPM/Yarn Scripts)

| Команда | Действие | Типичный сценарий |
| :--- | :--- | :--- |
| **`santa_auto`** | **🎯 Основная команда.** Запускает жеребьёвку и сборку для публикации. | Быстрый запуск для деплоя. |
| **`santa`** | Запускает только жеребьёвку получателей из `SANTA.csv`. | Новая жеребьёвка. |
| **`dev`** | Запускает локальный dev-сервер для просмотра. | Разработка и отладка интерфейса. |
| **`build`** | Собирает production-версию проекта. | Подготовка к публикации. |
| **`mock_auto`** | Генерирует тестовый CSV (15 записей) и запускает жеребьёвку. | Тестирование полного цикла. |
| **`mock_csv[ -- N]`** | Генерирует только тестовый CSV файл. | Создание данных для отладки. |
| **`stage_auto`** | Запрашивает текущий этап и собирает для публикации. | Смена этапа публикации. |
| **`stage[ -- N]`** | Устанавливает желаемый этап. | 1 = сбор, 2 = сервис, 3 = результаты |
