# 🎅 Random Santa

> Автоматическая жеребьёвка «Тайного Санты» с интеграцией Google Forms/Sheets и автодеплоем на GitHub Pages

[![Build and Deploy](https://github.com/vic-kk/random-santa/actions/workflows/build-and-deploy.yml/badge.svg)](https://github.com/vic-kk/random-santa/actions/workflows/build-and-deploy.yml)
![GitHub package.json version](https://img.shields.io/github/package-json/v/vic-kk/random-santa)

Проект решает главную проблему организатора «Тайного Санты»: сбор данных, исключение повторного выбора самого себя, честное распределение и публикация результатов — всё в одном месте, с минимальным ручным трудом.

**🚀 Живой пример:** [vic-kk.github.io/random-santa](https://vic-kk.github.io/random-santa/)

## ⚡ Быстрый старт

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/vic-kk/random-santa.git
cd random-santa

# Установить зависимости
npm install

# Режим разработки
npm run dev
```

## 📜 Скрипты
> [!TIP]
> Проект поддерживает npm и yarn.  
> Во всех примерах используется npm, для yarn заменяйте `npm run` -> `yarn`.

| Команда | Описание |
|---------|----------|
| `npm run dev` | Запуск dev-сервера |
| `npm run build` | Сборка проекта |
| `npm run lint` | Проверка кода ESLint |
| `npm run santa` | Запуск жеребьёвки (из CSV) |
| `npm run stage` | Интерактивная смена этапа проекта + сборка |
| `npm run stage -- 1 \| 2 \| 3` | Смена этапа проекта |
| `npm run santa_auto` | Жеребьёвка + переключение на финальный этап (3) |
| `npm run mock_csv` | Генерация (мока) тестового CSV (по умолчанию, 15 записей) |
| `npm run mock_csv -- N` | Генерация мока с N записями |
| `npm run mock_auto` | Генерация мока + жеребьёвка |
| `npm run preview` | Предпросмотр сборки |

## ✨ Функционал

- 🔑 **Уникальный ID** — генерируется автоматически и сохраняется в localStorage
- 📋 **Сбор данных через Google Form** — участники заполняют форму с адресами и пожеланиями
- 🎲 **Автоматическая жеребьёвка** — алгоритм гарантирует, что никто не дарит подарок сам себе
- 🎁 **Три этапа проекта**:
  - **Этап 1** — Сбор данных (показывается Google Form)
  - **Этап 2** — Технические работы (показывается сообщение об обслуживании)
  - **Этап 3** — Результаты (показываются данные получателя)
- 📋 **Копирование в буфер** — адреса можно скопировать одним кликом

## 🔄 Процесс работы (ручной)

### 1. Сбор данных
- Участники заполняют Google Form [пример формы](https://forms.gle/2z2TJb279D5bpKw9A)
- Форма собирает: ID, гендер, пожелания, адреса для доставки
- Данные экспортируются в CSV

### 2. Подготовка CSV
- Поместите файл `SANTA.csv` в папку `_local/`
- Формат CSV (пример):
```csv
"Отметка времени","Укажи уникальный номер, расположенный на сайте","Укажи свой гендер","Укажи свои пожелания, если хочешь. Что хотелось/не хотелось бы получить.","ОЗОН Адрес","ВБ Адрес"
"12/15/2025 10:18:29 GMT+7","610665","МУЖИК","Люблю сладкое! 🍫","Москва, улица Молодежная 60 корпус 5","Москва, улица Молодежная 60 корпус 6"
```

### 3. Жеребьёвка
```bash
npm run santa
```
Скрипт:
- Парсит CSV
- Проводит жеребьёвку (алгоритм derangement)
- Генерирует `src/data/addresses.ts` с результатами
- Создаёт бэкап предыдущей версии

### 4. Переключение этапов
```bash
# Этап 1: Сбор данных
npm run stage -- 1

# Этап 2: Обслуживание
npm run stage -- 2

# Этап 3: Результаты
npm run stage -- 3
```

### 5. Деплой
```bash
npm run build
```
Сборка создаётся в папке `docs/` для GitHub Pages.

## 🌐 Деплой на GitHub Pages (ручной)

1. Соберите проект:
```bash
npm run build
```

2. Закоммитите папку `docs/` и запушьте в репозиторий

3. В настройках репозитория GitHub:
   - Settings → Pages
   - Source: Deploy from a branch
   - Branch: main, folder: /docs


## 🌐 Автоматизация через GitHub Actions

В репозитории настроены несколько workflow:

1. **Автовыгрузка из Google Sheets с полным циклом перемешивания и смены этапа** — ручной запуск Action в Github, нужно настроить секрет (подробнее в `.github/workflows/`)
2. **Смена текущего этапа** — ручной запуск Action в Github
3. **Деплой на GitHub Pages** — при пуше в `main`

Для работы автовыгрузки потребуются GitHub Secrets:
- `CSV_SHEET_ID` — содержит id Google таблицы с ответами
